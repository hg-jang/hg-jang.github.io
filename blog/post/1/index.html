    기존 블로그 서비스를 놔두고 직접 만들게 된 이유

    많은 사람들이 이용하는 네이버, 티스토리 블로그 또는 개발자들의 비율이 특히 높은 velog를 놔두고 굳이 블로그를 직접 만들어 봤다. 나도 한 때는 velog, 네이버 블로그를 사용했었다. 이런 서비스들의 장점은 기록에만 집중 할 수 있다는 것이다. 문제는 나 같은 사람들은 쉽게 지친다는 것이다. 취업을 한 뒤에는 적응을 핑계로 한동안 여유롭게 지내며 여러 공부들을 조금씩 해봤는데, 이렇다보니 블로그는 오랫동안 방치되었다. 옛날에 살던 동네 앞에 10년이 넘도록 공사가 중단된 채 방치된 아파트 공사터가 있었는데 그 곳이 떠오른다.

    그동안 이런저런 고민도 많았는데, 새해를 맞이하며 점점 정리가 되어가고 있다. 동시에 개발 공부를 어떻게 해 나갈지 생각하다보니 하고 싶은게 많아졌다. 그래서 놀이터를 만들기로 했다. 굳이 기록에 연연하지 않고 만들고 싶은 것들을 이것저것 붙여나갈 수 있는 그런 장소. 그 과정과 결과를 상세히 남기면 더 좋겠지만 그것에 얽매이고 싶지는 않다.

    사람들은 현실에서 어떻게든 나만의 보금자리를 마련하려고 노력한다. 웹이 주무대인 나는 웹에서라도 나의 보금자리를 갖고 싶었던걸지도 모르겠다. 아무도 봐주지 않아도 되지만 돌아갈 장소가 있다는 것만으로도 위안이 되는.

    그래서 만들었다.

    계획대로 되지 않는다.

    누구나 그럴싸한 계획을 갖고 있다. 쳐맞기 전까지는.

    누가 한 말인지 몰랐는데, 마이크 타이슨이 한 말이란다. 맞는 말이다. 디자인한테 쳐맞고, aws한테 또 맞았다.

    촌스러운 디자인

    고슴도치도 제 새끼는 함함하다고 했던가. 돈주고 성형이라도 시켜주고 싶지만 내가 만들어서 나름 볼만한다. 나한테는....;;

    돈이 문제임

    프론트로 Next, 백으로 Nest를 선택하고 열심히 개발하였다. 어느 정도 프로토타입이 나오고 배포 테스트를 해보려고 했다. aws에서 프리 티어를 제공하는 ec2를 생성하고 postgresql, node를 설치하였다. 깃헙에서 코드를 가져와 build를 하는 순간 메모리 부족 알림이 떠서 당황했다. db랑 서버를 같은 인스턴스에 설치해서 그런건가 싶었다. 찾아보니 네스트만 돌려도 메모리 부족이 뜨는 경우가 있는 것 같아서 그냥 포기했다. 찾아올 사람도 없는데 돈주고 서버를 돌리기에는 아직인 것 같다.

    돈 좀 벌면 다시 도전...;

    그래서 Next만 사용해서 모든 페이지를 static하게 만들었다. api도 모두 없앴다. 이럴 줄 알았으면 도메인도 사지 말걸 그랬다.

    개발 과정에서

    지금까지 사용한 스택은 크게 다음과 같다.

    • Next.js
    • tailwindcss
    • typescript
    • Nest 마취 중
    • postgresql 마취 중

    Next

    Next를 찍먹해본 적은 있었는데 좀 더 깊게 다뤄보니 많이 신기했다. react에서 이렇게 정적인 페이지들을 만들어줄 생각을 어떻게 했을까? 진짜 이런거 만들어내는 사람들이 천재인가보다. 근데 css 시스템은 좀 불편한 것 같다. 내 기억으론 react에서 특정 컴포넌트나 페이지에서 css 파일을 임포트 해올 수 있었던 것 같은데 next에서는 글로벌 스타일을 제외하고는 모듈형으로 만들어서 임포트해야만 한다.

    tailwind

    되게 편한 것 같다. 클래스명과 스타일을 css 파일안에서만 관리하면 의존 관계를 찾아가는 것이 불편했는데 tailwind는 그런 점에서는 확실히 보기 수월한 것 같다. 특히 리액트처럼 컴포넌트 단위로 파일을 관리하는 프로젝트에서 말이다.

    또한 복잡한 셀렉터를 사용해야하는 경우에도 글로벌 스타일에서 커스텀 클래스명을 추가해서 만들면 되기 때문에 직관성이 떨어질 것 같지는 않다.

    typescript

    재밌다. 좋다.

    Nest

    Angular를 처음 써봤을 때 정형화된 구조가 인상적이었다. 개발을 길바닥에서 배워서 그런지 아직은 express처럼 자유로운 쪽보다는 nest, angular처럼 정형화된 툴이 좋은 것 같다. decolator, middleware, pipe, guard처럼 어떤 상황에서 어떤 것들을 써야하는지 알아가는 재미가 있었다. 근데 이젠 없어졌다.

    지금은 뼈맞고 침상에서 요양중이지만 시스템도 너무 거대해지기 전에 빨리 깨워야 할 것 같다.

    postgresql

    처음 백엔드를 배울 때 mysql을 배웠는데 그게 짱인 줄 알았다. 이제 postgresql이 짱인 것 같다. 사실 db에 관해서 아무것도 모르는 ㅈ밥이다. 회사에서 PostGIS를 자주 다루는데 openlayers는 map api를 사용해서도 이것저것 만들고 싶다.

    얘도 지금은 마취중

    앞으로 할 것들..

    메뉴 중에 업데이트 로그라는 것이 있다. 당분간 가장 활발하게 글이 올라갈 것이다. 적용해보고 싶은것이 많다. 다크모드 적용, 반응형, 테스팅, 다양한 ui/ux, 성능 개선 그리고 크고 작은 프로젝트들...

    앞서 말했지만 글로 하는 기록에 집착하지는 않을 것이다. 이 공간에서는 자유롭고 편안하고 싶다.

    일단 다음 업데이트는 아마 다크모드가 될 것이다. .
